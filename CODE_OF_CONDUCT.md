# Code of Conduct

Substrate is operator-grade. The conduct standard follows from that.

## What we expect

- **Disagree with the work, not the person.** Critique drafts, refusal patterns, claims, calibration. Avoid characterizing the operator behind them.
- **Cite or stop.** Anti-fabrication is the floor. Don't assert in an issue or PR what you wouldn't assert in a context file.
- **Operator voice over performative voice.** Direct. Specific. Falsifiable. No throat-clearing, no kill-list filler, no AI-glaze.
- **Falsifiability over plausibility.** A claim has to be checkable for it to count.
- **One change per PR.** Skill addition is one PR. Persona-fragment batch is one PR. Doc cleanup is one PR.

## What we won't tolerate

- Personal attacks, harassment, demographic stereotyping, or sustained hostility.
- Fabricated or inflated claims (in issues, PRs, discussions, or context files).
- Pressure on maintainers to relax the anti-fabrication or pre-publish gates.

## Enforcement

The repo maintainer ([Kesava Mandiga](https://github.com/k3sava)) handles conduct issues. Open a private GitHub security advisory or DM the maintainer via the GitHub profile with a short report, what happened, where, and what outcome you're asking for. Maintainer response within 7 days; faster if it's an active issue.

Outcomes range from a one-line note to a permanent ban. The maintainer's call.

## Scope

Applies to issues, pull requests, discussions, commit messages, and any communication clearly tied to Substrate (e.g. shared maintainer DM channels in the future).
