# Agent-first GTM (rebuild, don't bolt-on)

## Convergence
Four operators (and three Gartner forecasts) converge on the same structural claim: the GTM stack is not getting AI features, it is being rebuilt around AI. Agent-first GTM is a cross-functional pod with one mandate, a flywheel of named agents, and a "system of action" that replaces the seller's tab-stack. Treating it as a feature-add inside the current org is the failure mode.

## Operators
- Kieran Flanagan (HubSpot), `ins_rebuild-gtm-around-ai`. Build a cross-functional pod (product + eng + data + martech + ops) that owns AI-in-GTM as one mandate.
- Yamini Rangan (HubSpot), `ins_agent-first-gtm-flywheel` and `ins_reps-want-questions-not-dashboards`. Three-stage agent-first flywheel; reps don't want a better dashboard, they want to ask the deal questions.
- Claire Vo, `ins_agents-as-team-not-tools`. The same operating shape applied at the team level.
- Maja Voje, `ins_aeo-is-gtm-capability`. Answer Engine Optimization is a GTM capability, not an SEO experiment.
- Gartner, `ins_seller-workflows-begin-with-ai` (95% by 2027) and `ins_systems-of-action-replace-seller-stack`. Forecast and structural shift evidence.

## Variation
- Kieran focuses on *org structure* (single mandate pod).
- Yamini focuses on *flywheel design* and *seller experience*.
- Vo focuses on *team operating model*.
- Voje focuses on *channel capability* (AEO).
- Gartner provides the *macro forecast*.
- Convergence on direction; the variation is at which layer (org, flywheel, channel, forecast) the operator is intervening.

## Implication
If your AI-in-GTM plan is a project inside Marketing, RevOps, or Sales, it's the failure mode. The cross-functional pod with single mandate is the structural move. Within the pod, name the agents (per JTBD), build the system of action, and give reps natural-language deal-question access instead of a dashboard.
