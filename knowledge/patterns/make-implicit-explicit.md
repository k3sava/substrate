# Make the implicit explicit

## Convergence
Five operators converge on the same low-glamour move: write down what everyone "obviously knows", the decider, the criterion, the confidence level, the kill threshold, the company's operating mechanicals. This is the cheapest dysfunction-prevention tool in any room.

## Operators
- Claire Hughes Johnson (ex-COO Stripe), `ins_make-the-implicit-explicit` and `ins_company-house-foundation-beams-mechanicals`. Name the why, the decider, the criterion, the informed at the top of every meeting; install foundation, beams, mechanicals early.
- Annie Duke, `ins_make-intuition-explicit` and `ins_kill-criteria-make-pre-mortems-real`. Force intuitions into explicit predictions; pre-mortems only work with committed kill criteria.
- Mihika Kapoor (Figma), `ins_label-confidence-explicitly`. Label every strong opinion with a confidence level so the room knows when to push back.
- Aishwarya Naresh Reganti, `ins_transparency-in-uncertainty`. Show model uncertainty in the UI; opaque confidence destroys trust.

## Variation
- Hughes Johnson scopes it to *meetings + org operating system*.
- Duke scopes it to *decisions + forecasting*.
- Kapoor scopes it to *team voice + dissent design*.
- Reganti extends it to *AI product UX*, the same principle applied to model output.
- Shared: ambient knowledge is unreviewable; written knowledge is.

## Implication
Adopt one explicit-naming ritual per loop: the meeting opener (Hughes Johnson), the kill criterion (Duke), the confidence label (Kapoor), the uncertainty UI (Reganti). The cost is a sentence; the return is a room that can disagree productively.
